    Core

    Most of the core code is developed in a functional style, though the fr24.core.FR24 class provides a wrapper around them. Every response is automatically saved into the following directories:

    • Feed
      • feed/playback/{timestamp_ms}.parquet
      • feed/live/{timestmap_ms}.parquet
    • Flight history
      • flight_list/reg/{reg}.parquet, or
      • flight_list/flight/{iata_flight_num}.parquet
    • Playback
      • playback/metadata/{fr24_id}.parquet
      • playback/track/{fr24_id}.parquet
      • playback/track_ems/{fr24_id}.parquet

    Note that the fr24 flight ID is in base 10, not hex.

    The current implementation is poor and will be reworked in the future.

    Fetch metadata/history of flights for a given aircraft or flight number.

    Includes basic information such as status, O/D, scheduled/estimated/real times: see fr24.types.fr24.FlightData for more details.

    Use either reg or flight to query. To determine if there are more pages to query, check the response .result.response.page.more.

    Parameters:

    Name Type Description Default
    client AsyncClient

    HTTPX async client

    		 required
    reg None | str

    Aircraft registration (e.g. B-HUJ)

    		 None
    flight None | str

    Flight number (e.g. CX8747)

    		 None
    page int

    Page number

    		 1
    limit int

    Number of results per page (max 100)

    		 10
    timestamp int | datetime | Timestamp | str | None

    Show flights with ATD before this Unix timestamp

    		 'now'
    auth None | Authentication

    Authentication data

    		 None
